# ✅ Setup Complet - Hearst OS

**Date**: 23 avril 2026 06:55

## Configuration finale

### 🎯 Commande principale
```bash
npm run dev
```
**Comportement**: Kill tous les processus existants + nettoie les caches + redémarre TOUS les services (9000, 8100, 3000)

### ✅ Services opérationnels

| Service | Port | Status | Response | Log |
|---------|------|--------|----------|-----|
| **hearst-os** | 9000 | 🟢 | HTTP 307 | `/tmp/hearst-os.log` |
| **hearst-connect** | 8100 | 🟢 | HTTP 200 | `/tmp/hearst-connect.log` |
| **Hearst-app** | 3000 | 🟢 | HTTP 200 | `/tmp/hearst-app.log` |

### 📦 Modifications appliquées

#### 1. `package.json`
```json
{
  "name": "hearst-os",
  "scripts": {
    "dev": "./launch.sh",           // ← Lance TOUT
    "dev:solo": "next dev -p 9000", // ← Hearst-os uniquement
    "dev:fresh": "kill + clean + dev:solo",
    "stop": "./launch-stop.sh",
    "launch": "./launch.sh"
  }
}
```

#### 2. Scripts créés
- `launch.sh` - Kill + clean + démarre tout (9000, 8100, 3000)
- `launch-stop.sh` - Arrête tous les services
- Permissions: `chmod +x` appliqué

### 🚀 Workflow de développement

```bash
# Démarrer (kill + redémarre tout)
npm run dev

# Arrêter tous les services
npm run stop

# Voir les logs en temps réel
tail -f /tmp/hearst-os.log        # Frontend principal
tail -f /tmp/hearst-connect.log   # Backend connexion
tail -f /tmp/hearst-app.log       # Landing page

# Démarrer uniquement hearst-os
npm run dev:solo

# Clean restart hearst-os uniquement
npm run dev:fresh
```
