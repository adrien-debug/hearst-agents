Heuristique phase courante

Faute d'API "phase nommée" exposée par le runtime, la phase courante est dérivée de events.filter(e => e.type === "step_started").length % 5. Pour les états spéciaux : awaiting_approval → phase 5 (deliver gating), awaiting_clarification → phase 1 (intent attend un input). Quand le runtime exposera une vraie séquence de phases nommées, swap l'heuristique par la source réelle (1 ligne).
