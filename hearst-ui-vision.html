🔄 Flow d'Interaction — De IDLE à Asset Créé

1
🎯 État IDLE — Attente d'intention
L'utilisateur ouvre l'app. Le chat flotte au centre. Halo pulsé doucement. Suggestions contextuelles basées sur l'historique et l'heure.
Visuel : Halo lumineux + Input flottant + 4 suggestions chips
Action utilisateur : Clic sur suggestion ou tape dans l'input
2
✍️ Input Utilisateur → Détection
L'utilisateur saisit : "Analyse mes ventes Stripe de mars"
Détection automatique : domaine = Finance, connecteur = Stripe, action = Analyse
Feedback immédiat : Halo s'intensifie (animation) + Typing indicator
SSE Event : {type: "intent_detected", domain: "finance", connector: "stripe"}
3
🤖 Agent Spécialisé Activé
Finance Agent prend le relais. Backend sélectionné (OpenAI pour calculs). Le chat descend en bas (anchored). Focal Object apparaît en cours de création.
Transition : Chat glisse vers le bas (300ms)
Focal : Skeleton loader avec "Préparation de l'analyse..."
Halo : Pulsation rapide (état "processing")
4
📊 Exécution & Streaming
L'agent exécute : appelle Stripe → récupère données → OpenAI analyse → génère rapport.
Chaque étape est streamée vers l'UI en temps réel.
SSE Events :
{type: "step", tool: "stripe", status: "running"}
{type: "step", tool: "stripe", status: "done"}
{type: "thinking", content: "Analyse des patterns de vente..."}
UI : Steps apparaissent dans le Focal + Timeline
5
✅ Asset Généré & Manifestation
Le rapport est prêt. Apparition complète du Focal Object avec le contenu. L'ancien Focal descend dans la Timeline. Le nouveau devient le centre d'attention.
Animation : Fade-in du contenu (500ms) + Glow cyan
CTA : "Télécharger PDF", "Partager", "Créer une mission"
Right Panel : Nouvel item en top de Timeline + Focal Context mis à jour
6
🔄 Suite ou Nouvelle Action
L'utilisateur peut :
• Poser une question sur le rapport (chat en bas)
• Télécharger/partager (CTA)
• Créer une mission récurrente ("Fais ça tous les lundis")
• Nouvelle requête (efface et recommence)
Chat : Input réactivé en bas avec contexte du Focal actif
Suggestion : "Créer une mission hebdomadaire ?" apparaît dans suggestions

🤖 Missions — Création depuis le Chat

1
Création d'une Mission depuis un Asset
Après avoir généré un rapport, l'utilisateur clique "Créer une mission".
Ou demande directement : "Fais ça tous les lundis matin"
CTA : Bouton "🔔 Créer une mission" sous chaque Asset
Chat : "Répète cette analyse tous les vendredis"
2
Configuration Intelligente
L'agent détecte la fréquence, les paramètres, et les conditions.
Propose une configuration, l'utilisateur valide ou modifie.
Modal : "Configuration de la mission"
• Nom : "Rapport Hebdo Stripe"
• Fréquence : "Tous les vendredis 9h00"
• Action : "Générer + Envoyer email à team@company.com"
• Condition : "Seulement si ventes > $10K"
3
Activation & Monitoring
La mission apparaît dans le Right Panel. Toggle ON/OFF.
Prochaine exécution affichée. Historique des runs accessible.
Right Panel : Section "Missions Auto"
• Toggle actif/inactif
• Prochain run : "Vendredi 9h00 dans 2 jours"
• Dernier run : "✅ Réussi il y a 5 jours"

🎭 États de l'Interface

IDLE — Attente

◉
  • Halo pulsé lentement
  • Chat flottant au centre
  • Suggestions contextuelles
  • Timeline visible mais passive
  • Pas de Focal Object

ACTIVE — Exécution

◉
  • Halo pulsation rapide
  • Chat ancré en bas
  • Focal Object visible (loader puis contenu)
  • Steps visibles en temps réel
  • Input actif pour questions

🎯 Principes Fondamentaux

💬

Chat-First

Tout passe par le chat. Pas de boutons partout, pas de navigation complexe. L'utilisateur parle, HEARST exécute.

👁️

Manifestation

Les résultats apparaissent dans le Focal Object. Pas de navigation, pas de pages à charger. Instantané, contextuel.

🤖

Automatisation

Tout Asset peut devenir Mission. Un clic, une phrase. "Fais ça tous les lundis" → c'est programmé.

🧠

Invisible

L'utilisateur ne voit pas l'agent, ni le backend, ni les connecteurs. Il voit juste le résultat. L'OS pense, l'utilisateur agit.
