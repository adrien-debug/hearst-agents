❌ Les vrais problèmes bloquants

  • 🔴

    Le LLM ne reçoit pas les données

    Le system prompt dit "tu peux accéder" mais aucune donnée réelle n'est injectée. Résultat : "Je ne peux pas accéder à votre calendrier" malgré la connexion.

  • 🔴

    Les missions automatiques ne s'exécutent pas

    Le scheduler détecte les patterns "tous les matins" mais la création réelle de mission depuis le chat est inactive.

  • 🔴

    Focal objects avec mocks uniquement

    Les focal objects affichent des données de démo, pas les vraies données utilisateur.

🎯 Vision Proposée

HEARST OS doit être un OS de productivité conversationnel où l'utilisateur gère sa bureautique (emails, calendrier, tâches, documents) via des dialogues naturels avec des agents spécialisés.

Core Value Proposition

"Parle à ton travail comme tu parles à ton assistant. HEARST comprend, découpe, délègue aux bons agents, et te présente le résultat dans une interface focalisée."

Non-Scope (Ce qu'on ne fait PAS)

  • 🚫

    Pas d'infrastructure cloud (OpenCloud)

    Gérer AWS/GCP/Azure via HEARST = trop complexe, pas le core business productivité.

  • 🚫

    Pas de messaging multi-canal (OpenClaw)

    WhatsApp/Telegram/Signal via HEARST = complexité ops énorme pour un use case marginal.

  • 🚫

    Pas de CRM/Finance avancé

    Stripe/HubSpot basiques OK, mais pas de remplacement Salesforce/NetSuite.

🗓️ Roadmap Proposée

Phase 1 : Core Fonctionnel Semaines 1-3
  • Récupération réelle des events Google Calendar et injection dans le contexte LLM
  • Récupération réelle des emails Gmail et résumé intelligents
  • Création automatique de missions depuis le langage naturel
  • Fix : "Je ne peux pas accéder" → "Voici tes 5 meetings d'aujourd'hui..."
Phase 2 : Focal Objects Actifs Semaines 4-6
  • Focal objects avec vraies données (pas de mocks)
  • Actions interactives : approve, reject, snooze, delegate
  • Morphing des objects (plan → draft → final → archived)
  • Right Panel contextuel et utile
Phase 3 : Beta Privée Semaines 7-10
  • Onboarding 10 utilisateurs beta
  • Feedback loop : observabilité complète des usages
  • Fix des bugs edge cases
  • Documentation utilisateur et guides
Phase 4 : Public Launch Semaines 11-14
  • Pricing tiers : Free / Pro ($19) / Team ($49)
  • Site marketing et vidéo de démo
  • Support et documentation complète
  • Product Hunt / BetaList launch
