Ce qu'il reste a faire

Le plus simple est de voir la suite comme 4 blocs clairs.

1. Finir le shell reel

Zero drift sur `app/(user)/*`, scroll/messages propres, hierarchie centre/rail totalement nette, docs coherence totale.

2. Cleanup runtime / API

Retirer les doubles verites, clarifier les chemins legacy, solidifier les contrats reels du runtime et des routes.

3. Ouvrir les gros chantiers absents

`platform`, settings, RBAC, assets cloud, `asset_files`, connector packs, admin dynamique.

4. Gate final

Lint, build, tests, smoke complet et verification finale contre la vision.

Lecture simple: le coeur produit est deja la. La vraie distance a la vision finale se situe surtout dans la finition stricte du shell, puis dans la grande convergence d'architecture `lib/`.
