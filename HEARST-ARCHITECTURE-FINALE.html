Assets & Stockage

Où Mettre les PDF et Fichiers Générés

Architecture de stockage multi-tier : local (dev) → cache (hot) → cloud (cold).

1

Création de l'Asset

PDF/Excel généré → lib/engine/runtime/assets/create.ts
Factory avec tenantId, workspaceId, runId

2

Stockage Hybride (Hot + Cold)

Tier 1 (Hot): Local filesystem .runtime-assets/ — Accès immédiat
Tier 2 (Cloud): R2/S3 — Persistance durable, CDN edge

3

Métadonnées en DB

Table asset_files : storage_kind, bucket, key, url_signée, checksum, expires_at

4

Cache & Distribution

Redis L1 pour metadata. CDN pour fichiers. URL signées 1h pour sécurité.

5

Cleanup Automatique

Worker cron quotidien. TTL 30j par défaut. Archivage Glacier après 90j.

Type d'Asset Générateur Stockage Cache TTL
PDF Report assets/generators/pdf.ts Hot + Cloud Redis (1h) 30 jours
Excel Export assets/generators/spreadsheet.ts Hot + Cloud Redis (1h) 30 jours
Image/Thumbnail assets/generators/image.ts CDN Edge CDN (24h) 7 jours
JSON Data Inline DB Only Redis (5min) —
Mission Config Inline DB Only Redis (1h) —
💡 Clé de l'Architecture : L'abstraction StorageProvider permet de switcher entre local et cloud sans changer le code métier. Développement local → filesystem. Production → R2/S3 automatiquement via env vars.
Services & Connecteurs

Où Mettre les 200+ Intégrations

Pattern Connector Pack : un pack = un domaine métier = 25+ connecteurs cohérents.

📦

Connector Pack Pattern

Chaque pack contient : manifest.json (capabilities), auth/ (OAuth), services/ (API wrappers), mappers/ (→ Unified types), schemas/ (Zod).

🔌

Auto-Discovery

Les packs sont scannés au boot. Manifest.json déclare capabilities, auth type, dépendances. Registration automatique.

🛡️

Health & Resilience

Chaque connecteur expose healthCheck(). Circuit breaker intégré. Fallback à cache automatique si API down.
