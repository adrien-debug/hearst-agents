- [ ] Session persistence (reload page)

### 2. CHAT & CONVERSATIONS (10 tests)
Créer 10 conversations avec:
- [ ] **Conv 1**: Message simple "Hello"
- [ ] **Conv 2**: Question avec réponse courte
- [ ] **Conv 3**: Question contextuelle (référence message précédent)
- [ ] **Conv 4**: File search "cherche mes documents"
- [ ] **Conv 5**: Code "calcule fibonacci"
- [ ] **Conv 6**: Email (si Gmail connecté) "résume mes emails"
- [ ] **Conv 7**: Calendar (si Calendar connecté) "mes réunions"

**Validation par conversation:**
- Thread créé en DB
- Messages persistés (user + assistant)
- SSE events reçus (text_delta, orchestrator_log, done)
- Response cohérente

### 3. RUNTIME/RUNS (10 tests)
Exécuter 10 runs via `/api/orchestrate`:
- [ ] **Run 1**: Mode direct_answer (question simple)
- [ ] **Run 2**: Mode plan_and_execute (tâche multi-étapes)
- [ ] **Run 3**: Mode managed_agent (avec agent spécifique)
- [ ] **Run 4**: Avec focal_context (document)
- [ ] **Run 5**: Avec mission_id (scheduled)
- [ ] **Run 6**: Backend V2 + Responses API
- [ ] **Run 7**: Backend V2 + Assistants API
- [ ] **Run 8**: Backend V2 + Handoff (si possible)
- [ ] **Run 9**: Erreur gérée (timeout/invalid input)
- [ ] **Run 10**: Run complet avec tool call

**Validation par run:**
- Run ID généré
- Events SSE complets (run_created → run_completed/failed)
- Timeline events persistés
- Métriques: tokens, cost, latency
- Status final correct

### 4. MISSIONS (10 tests)
Créer et exécuter 10 missions:
- [ ] **Mission 1**: One-shot immédiat (run now)
- [ ] **Mission 2**: Schedule quotidien

- [ ] **Mission 5**: Longue durée (multi-steps)

**Validation par mission:**
- Mission créée en DB

- Logs d'exécution

**Validation par action:**

### 6. ASSETS (2 créations)
Créer 2 assets:
- [ ] **Asset 1**: Document type (markdown/text)

  - Provenance: "audit_test"

## 📝 Notes pour l'Agent

### Commandes Utiles
```bash
# Test API orchestrate
curl -X POST http://localhost:9000/api/orchestrate \
  -H "Content-Type: application/json" \
  -d '{"message":"hello","surface":"home"}'

# Test missions
curl http://localhost:9000/api/v2/missions

# Test runs
curl http://localhost:9000/api/v2/runs

# Vérifier DB
npm run db:console  # ou supabase console
```

### Endpoints de Test
- `POST /api/orchestrate` — Chat principal
- `POST /api/v2/missions` — Créer mission
- `POST /api/v2/missions/[id]/run` — Run mission
- `GET /api/v2/runs` — Liste runs
- `GET /api/v2/right-panel` — UI aggregate

### Stores Zustand (dev tools)
- Ouvrir DevTools → Application → Storage
- Vérifier stores persistant bien rechargé
